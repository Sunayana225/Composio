🔑 Key Insights

→

Auth is the #1 integration cost. OAuth2 apps need token refresh, scope negotiation, consent screens. API Key apps are simpler but less secure. The toolkit needs an auth abstraction layer.

→

Developer/infra apps are the easiest wins. GitHub, Stripe, Supabase, Notion — all have OAuth2, free tiers, broad APIs. First 10 integrations should be these.

→

Enterprise apps need outreach, not code. Salesforce Commerce Cloud, DealCloud, PitchBook — they have APIs but need a partnership. These are relationship plays.

→

MCP is nascent (13%). Only 13 of 100 have MCP servers. The early-mover advantage: wrap the other 87 with agent toolkits before they build their own.

→

Marketing/ads is the most gate-heavy category. Meta Ads, LinkedIn Ads, Google Ads all have multi-week app review processes. Production access needs patience.

→

API surface correlates with company maturity. Salesforce, Stripe, GitHub, Twilio have 200+ endpoints. Niche/startup apps have 1-20. Prioritize by API breadth.

📄 Methodology & Data Notes ▼

Research method: Each app was analyzed using AI training knowledge (Claude Sonnet) covering developer documentation, API references, and technical blogs. Less-common apps were flagged with low/medium confidence.

Auth methods: The primary auth method is listed in the table. Many apps support multiple methods; the full list appears in each row's detail view.

Self-serve: "Yes" = developer can get API credentials with a free account/trial. "Partial" = needs paid plan or approval. "No" = enterprise partnership required.

API breadth: Narrow (1-10 endpoints), Medium (10-50), Broad (50+). This is approximate — some broad APIs (Salesforce, Stripe) have 200+.

MCP status: "Yes" = official or well-maintained community MCP server exists. "No" = none found. Absence of MCP is the biggest opportunity for Composio.
