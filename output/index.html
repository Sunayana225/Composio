Key Insights

→

Auth is the #1 integration cost. OAuth2 apps need token refresh, scope negotiation, consent screens. API Key apps are simpler but less secure. The toolkit needs an auth abstraction layer.

→

Developer/infra apps are easiest wins. GitHub, Stripe, Supabase, Notion — all have OAuth2, free tiers, broad APIs. First 10 integrations.

→

Enterprise apps need outreach, not code. Salesforce Commerce Cloud, DealCloud, PitchBook — they have APIs but need partnerships.

→

MCP is nascent (13%). Only 13 of 100 have MCP servers. The early-mover advantage: wrap the other 87.

→

Marketing/ads is the most gate-heavy. Meta Ads, LinkedIn Ads, Google Ads all have multi-week app review processes.

→

API surface correlates with maturity. Salesforce, Stripe, GitHub, Twilio have 200+ endpoints. Niche/startup apps have 1-20.

Methodology & Data Notes

Research method: Each app was analyzed using AI training knowledge (Claude Sonnet) covering developer documentation, API references, and technical blogs. Lesser-known apps flagged with low/medium confidence.

Auth: Primary method shown in table. Full auth details appear in each row's detail view. Many apps support both OAuth2 and API Key.

Self-serve: "Yes" = free account/trial with API access. "Partial" = needs paid plan or approval. "No" = enterprise partnership required.

API breadth: Narrow (1-10 endpoints), Medium (10-50), Broad (50+). Broad APIs like Salesforce and Stripe have 200+.

MCP: "Yes" = official or well-maintained community MCP server. This is the biggest opportunity for Composio.
